DO 服务器自建AI服务（划水）：从 DigitalOcean+New API+Open WebUI+CLIProxyAPI

发表于2026-04-05|更新于2026-04-05
|浏览量:

本文记录了从零开始，在 DigitalOcean 服务器上部署一套完整 AI 自用服务的全过程，包括服务器购买与初始化、1Panel 面板安装、New API 部署、Open WebUI 部署、CLIProxyAPI 部署，以及关键的踩坑与修复记录。

一、为什么选择 DigitalOcean

起因是发现自己手头有 GitHub 学生包，其中包含 DigitalOcean 的 $200 美元额度（有效期 1 年）。算了一下：

  • $18/月 的 2 vCPU + 2GB 内存 + 60GB SSD 套餐
  • 198，基本榨干额度
  • 比 $12/月 的 1 核机器多了一个 CPU 核心，对 Docker 多服务并发更友好

⚠️ 注意：DigitalOcean 即使关机也收费，不用的实例一定要 Destroy（彻底销毁），否则仍然计费。

关联 GitHub 学生包

  1. 前往 GitHub Education 找到 DigitalOcean 福利
  2. 在 DigitalOcean 的 Billing → Promo Code 页面输入优惠码
  3. 如果找不到优惠码入口，可以尝试访问 https://www.digitalocean.com/github-students，或直接提交工单（Billing 类型）说明情况，客服处理很快

二、创建服务器（Droplet）

在 DO 控制台点击 Create → Droplet，配置如下：

  • Region：旧金山（SFO，离国内延迟相对可接受）
  • Image：Ubuntu 22.04 LTS
  • Plan：Basic → Premium Intel/AMD，$18/月（2 vCPU / 2GB / 60GB）
  • SSH Key：提前在本机生成公钥并上传到 DO 账户

💡 DO 控制台 Settings → Security 里保存的 SSH Key 只在新建 Droplet 时自动注入，已有的机器需要手动写入 ~/.ssh/authorized_keys。

四、域名配置：用 Cloudflare 管理所有子域名

本文使用 Cloudflare 管理域名 DNS，所有服务通过子域名 + 1Panel 反向代理方式对外提供 HTTPS 访问。

4.1 基本原理

浏览器访问域名时默认只走 443（HTTPS）或 80（HTTP）端口。通过 OpenResty（Nginx 增强版） 作为"门卫"：

  • 占用 80/443 端口，统一接收所有流量
  • 根据访问的域名，将流量转发到内部对应的端口
  • 同时统一处理 HTTPS 证书

4.2 安装 OpenResty

在 1Panel 的 应用商店 中搜索 OpenResty，一键安装，保持默认配置（HTTP 80、HTTPS 443），点击确认即可。

4.3 以面板域名为例（panel.your-domain.com）

Step 1：Cloudflare 添加 DNS 记录

字段 值
Type A
Name panel
IPv4 your.server.ip
Proxy 先关闭（灰色云朵）

先关闭 Proxy 是为了让 Let's Encrypt 能通过 HTTP 验证来颁发证书。证书申请完成后再开启。

Step 2：1Panel 申请 HTTPS 证书

进入 网站 → 证书 → 创建证书，填入 panel.your-domain.com，验证方式选 HTTP，申请免费的 Let's Encrypt 证书。

Step 3：1Panel 创建反向代理

进入 网站 → 网站 → 创建网站 → 反向代理：

  • 主域名：panel.your-domain.com
  • 代理地址：http://127.0.0.1:<YOUR_PANEL_PORT>

保存后进入该网站的 HTTPS 配置，选择刚申请的证书，开启 HTTPS 并勾选强制 HTTPS。

Step 4：回到 Cloudflare 开启橙色云朵（Proxied）

这样真实 IP 会被隐藏，流量经过 Cloudflare CDN，同时确保 SSL/TLS 模式设置为 Full（严格），否则可能出现重定向循环。

此后所有子域名的配置步骤完全相同，只需修改 Name 和代理端口。

5.5 踩坑：从 Windows 本地版迁移数据

如果你之前在 Windows 上用 MSI 安装包安装的 New API，数据保存在安装目录下的 new-api.db 文件（约 1MB+）。

迁移步骤：

  1. 关闭 Windows 上的 New API 程序
  2. 找到 new-api.db 文件（右键快捷方式 → 打开文件所在位置）
  3. 上传到服务器的 /opt/1panel/apps/new-api/data/

关键坑点： 不同版本的镜像读取的数据库文件名不同，有的读 new-api.db，有的读 one-api.db。最稳妥的做法是复制两份：

7.7 接入 New API

进入 New API 渠道管理，添加新渠道：

  • 类型：OpenAI 或自定义
  • 代理地址：http://172.17.0.1:8317

⚠️ 重要踩坑：不能填 http://127.0.0.1:8317！

原因：New API 跑在 Docker 容器内，127.0.0.1 指向的是容器自身，而不是宿主机。容器之间访问宿主机上的其他服务，需要用 Docker 默认网关 IP 172.17.0.1，或者直接用已配置的域名（如 https://proxy.your-domain.com）。
