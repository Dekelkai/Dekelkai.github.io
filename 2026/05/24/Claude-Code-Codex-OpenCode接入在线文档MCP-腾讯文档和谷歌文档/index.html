在 Claude Code、Codex、OpenCode 中接入在线文档 MCP：腾讯文档和谷歌文档

发表于2026-05-24|更新于2026-05-24|技术教程
|浏览量:

这篇记录一个很实用的工作流。

现在 Claude Code、Codex、OpenCode 这类 AI 编程工具已经不只是能读本地代码了。

通过 MCP，它们还可以接入外部服务。

比如腾讯文档。

比如谷歌文档。

这样就可以把需求文档、会议记录、表格数据和代码仓库串起来，不用再一边复制文档内容，一边粘到 AI 工具里。

这件事很小，但体验上会顺很多。

什么是 MCP

MCP，全称是 Model Context Protocol。

它是 Anthropic 发起的一个开放协议，用来让 AI 助手连接外部工具、数据库、文件系统和云服务。

以前你想让 AI 看一份腾讯文档，通常要先打开网页，复制内容，再粘贴到对话里。

现在有了 MCP，理论上你可以直接说：

或者：

也就是说，MCP 解决的是「AI 怎么安全、标准地连接外部工具」这个问题。

本文只讲在线文档这一类场景。

三个工具的配置差异

Claude Code、Codex、OpenCode 都支持 MCP，但配置格式不一样。

简单对比如下：

工具 配置文件 传输类型 Header 支持
Claude Code ~/.claude/ stdio / HTTP 支持 --header 参数
Codex ~/.codex/config.toml stdio / HTTP 支持 bearer_token_env_var / http_headers
OpenCode ~/.config/opencode/opencode.json local / remote 支持 headers 字段

核心思路其实一样。

如果是远程 HTTP MCP，就配置 URL 和鉴权信息。

如果是本地 stdio MCP，就配置启动命令，比如 node /path/to/server.js。

差别主要在配置文件格式。

腾讯文档 MCP 能做什么

腾讯文档官方提供了 MCP 接入能力。

能做的事情大致分几类。

读取类：

  • 获取文档列表和最近查看列表
  • 读取文档内容，比如 Word、Excel、智能表格、智能文档
  • 搜索文档和空间

创建类：

  • 创建智能文档 SmartCanvas
  • 创建 Excel
  • 创建 PPT
  • 创建思维导图
  • 创建流程图
  • 创建 Word

编辑类：

  • 编辑文档内容
  • 批量更新表格单元格
  • 管理智能文档里的页面、文本、标题、待办事项
  • 管理智能表格字段和记录

空间管理类：

  • 查询空间节点
  • 创建空间节点
  • 删除空间节点
  • 管理文件夹结构

对个人使用来说，最常见的场景就是让 AI 读取需求文档，然后结合代码仓库做开发。

或者让 AI 把阶段性分析结果写回在线文档。

谷歌文档 MCP 的三种方案

谷歌文档目前可以按需求选择三类方案。

如果你想功能完整，优先看 Google 官方 Workspace MCP。

如果你想本地私有部署，可以看开源 google-docs-mcp。

如果你不想折腾 OAuth，可以看 Composio 托管方案。

这是功能最完整的方案。

怎么选择方案

场景 推荐方案
腾讯文档，三种工具都要用 腾讯文档官方 MCP
谷歌文档，想要完整功能 Google 官方 Workspace MCP
谷歌文档，不想折腾 OAuth Composio 托管方案
谷歌文档，需要本地私有部署 开源 google-docs-mcp

我的建议是，国内文档优先用腾讯文档官方 MCP。

它的接入路径很短，基本就是拿 Token，然后在工具里配置 HTTP MCP。

谷歌文档则看你的使用场景。

如果你只是偶尔用，Composio 更省事。

如果你要长期重度使用，Google 官方 Workspace MCP 更值得折腾。

常见错误处理

错误 原因 解决方案
腾讯文档 400006 Token 鉴权失败 检查 Header key 必须是 Authorization
腾讯文档 400007 权限不足 检查账号权限或服务权限
Codex stdio 无法启动 路径或命令错误 先在终端直接运行命令验证
OpenCode HTTP 不生效 配置类型不匹配 HTTP MCP 使用 type: remote
本地 google-docs-mcp 授权异常 credentials.json 或 token.json 问题 重新检查 OAuth 文件和授权流程

安全注意事项

这类教程最容易犯的错误，就是把真实 Token 写进文章或者仓库。

建议统一遵守几条规则：

  • 公开文章只写占位符，比如 你的腾讯文档Token
  • 私钥、Token、API Key、Client Secret 不截图
  • 项目级配置如果包含敏感信息，就不要提交
  • 本地 OAuth 文件加入 .gitignore
  • 能用环境变量就尽量用环境变量

小结

三大工具都能接 MCP。

Claude Code 的优势是命令行体验简单，claude mcp add 一行就能加。

Codex 的配置是 TOML，适合用 bearer_token_env_var 管理 Token。

OpenCode 的配置是 JSON，type: remote 对应 HTTP MCP，type: local 对应本地 stdio MCP。

腾讯文档目前接入体验很直接，适合先试。

谷歌文档则可以根据需求选择官方 Workspace MCP、开源本地 MCP 或 Composio 托管方案。

把在线文档接进 AI 编程工具之后，很多工作流会变得自然很多。

需求在文档里，代码在仓库里，AI 可以同时看两边。

这才是 MCP 真正有意思的地方。
