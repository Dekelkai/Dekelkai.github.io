Windows 使用 SSHFS 远程挂载服务器目录教程

发表于2026-05-22|更新于2026-05-22|技术教程
|浏览量:

背景

在多人共享的服务器环境中开发时，经常会遇到这样的问题：

  • 不想和其他用户的配置混在一起
  • 希望使用本地的开发工具配置（如 Claude Code、VS Code 插件等）
  • 需要编辑服务器上的文件，但又想保持本地的开发体验

本文介绍如何使用 SSHFS-Win 在 Windows 上挂载远程服务器目录，实现"本地配置 + 远程文件"的开发模式。

方案对比

方案 A：VS Code Remote-SSH

工作原理：VS Code 在远程服务器上安装 VS Code Server，扩展和命令在远程执行。

优点：
- 文件操作自然流畅
- 终端命令直接在服务器执行

缺点：
- 使用的是服务器上的配置（~/.claude、~/.vscode-server 等）
- 多人共享服务器时容易配置冲突

方案 B：SSHFS 挂载（本文推荐）

工作原理：将服务器目录映射为 Windows 本地盘符，VS Code 和开发工具在本地运行。

优点：
- 使用本地 Windows 配置
- 不会和服务器其他用户冲突
- 可以同时挂载多个服务器目录

缺点：
- 大项目可能有延迟
- 命令需要通过 SSH 手动执行

总结

SSHFS-Win 方案适合以下场景：

  • 多人共享服务器，不想配置冲突
  • 需要使用本地开发工具配置
  • 主要进行文件编辑，命令执行频率不高

如果你的工作流是频繁执行服务器命令、需要实时查看日志，建议使用 VS Code Remote-SSH 或配合 1Shell 等工具使用。
